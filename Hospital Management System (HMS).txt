System Overview

* This system is a role-based hospital management platform with four main user roles:
Admin
Doctor
Receptionist
Patient
The application is API-driven, consumed by web/mobile clients.

* User Roles & Responsibilities

1. Admin
Default admin account exists in DB (created at startup).
Can login & change password.
Can create:
Doctor
Receptionist
Generates unique registration codes for doctors & receptionists.
Sends registration email with code.
Can view all users.

2. Doctor
Registers using:
Email
Password
Unique registration code
After login:
View & update profile
Add specialization & experience
View appointments
Add prescriptions
View patient medical history
Mark appointment as completed

3. Receptionist
Registers using:
Email
Password
Unique registration code
Can:
View doctors & their schedules
Add appointments on behalf of patients
Confirm appointments physically or via call

4. Patient
Self registration (email + password)
Login
View doctors by specialization
View doctor availability
Book appointment
Make payment (before or after visit)
View prescriptions & medical history
